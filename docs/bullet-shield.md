# Bullet Shield 简介

> 来源：[RoboWiki / Bullet Shielding](https://robowiki.net/wiki/Bullet_Shielding)、[RoboWiki / Shield](https://robowiki.net/wiki/Shield)。
>
> 在 Robocode 经典版 1v1 中，**bullet shield**（子弹盾 / 挡弹）是一种主动防御战术：
> 朝敌方子弹的来向发射一颗低威力子弹，让两颗子弹在空中相撞并互相抵消，
> 从而避免自身受到伤害。

## 1. 基本原理

Robocode 里子弹相撞的处理规则是：

- 任意两颗子弹的**碰撞箱**（一个以子弹当前位置为中心的小圆）发生重叠时，
  两颗子弹都会**立刻消失**，谁也不造成伤害。
- 子弹威力越低，碰撞箱半径越小；威力越高，半径越大。

因此，只要自己发射的子弹能“挡”在敌方子弹的飞行路径上，
就可以用一颗代价很小的低威力子弹换走敌方一颗 potentially 高伤害子弹。

```text
敌方子弹 ──────────────►
                         相撞点
己方子弹 ◄───────────────
```

## 2. 为什么需要专门做 shield

普通 1v1 机器人的防御思路是：

- **躲避**：通过 wave surfing、随机运动等让对方打不中。
- **距离控制**：拉远或贴近让对方命中率下降。

bullet shield 则提供了一条完全不同的思路：

- **不需要大幅机动**，只要炮口能对准来弹方向即可。
- 对低射速、高威力、轨迹可预测的对手效果尤为明显。
- 在狭窄地形或自己处于劣势时，可以显著延长存活时间。

## 3. 实现的关键难点

真正稳定地 shield 并不容易，主要挑战有：

### 3.1 精确知道敌方子弹何时出现、飞向哪里

没有雷达事件直接告诉你“敌方开火了”。通常需要从扫描数据中推断：

- 敌方能量突然下降某个 `power` 值（`0.1 ≤ power ≤ 3.0`），就说明它开火了。
- 从敌方位置、能量下降量、自身位置，可以重构出子弹的起点、方向、速度和威力。
- 这就是 wave tracking 的核心。

### 3.2 拦截窗口非常窄

敌方子弹速度 `20 − 3 × power` 很快；自己发射的低威力子弹速度也很快。
两颗高速小圆要在空间和时间上对齐，容错极低：

- 己方发射时机晚一回合，可能就错过。
- 炮口方向差几度，子弹就会从敌方子弹旁边飞过。
- 自己移动时炮口坐标也在变，需要预测自身位置。

### 3.3 敌方不会乖乖站着让你 shield

高阶对手会：

- 快速变换角度或距离，让你的 shield 计算失准。
- 使用低威力、高射速子弹，提高你的拦截密度要求。
- 故意朝你炮口盲区开火。

## 4. 常见的 shield 形式

| 形式 | 做法 | 特点 |
|------|------|------|
| **被动 shield** | 只在检测到敌方 wave 时才发射拦截弹 | 省能量，但窗口要求极高 |
| **主动 shield** | 持续朝敌方发射低威力子弹，形成“弹幕” | 能量消耗大，但对可预测射击很有效 |
| **混合 shield** | 正常瞄准射击，同时在敌方开火瞬间补一颗 shield 弹 | 实现复杂，攻防兼顾 |

## 5. 反制 shield：为什么 Ronin 要打边缘

Bullet shield 的弱点在于：**拦截点集中在敌方与自己之间的连线上**。
如果子弹不是沿着这条“直接连线”飞行，拦截难度会急剧上升。

因此，当 Ronin 的 `EnemyTracker` 判断对手很可能在 shield 时，
`ShieldAimSelector` 会切换瞄准策略：

- 不再瞄准敌方车身中心。
- 而是瞄准敌方**边缘**（left edge / right edge），让子弹轨迹偏离直接连线。
- 这样敌方即便想做 shield，也很难有足够的碰撞箱把子弹拦下。

这种反制手段属于**检测 + 瞄准偏移**的组合，不改变 Ronin 的整体射击模型。

## 6. 设计权衡

### 6.1 shield 的收益

- 抵消敌方高威力子弹，避免大量失血。
- 不需要依赖复杂运动，适合炮台机动强、车身机动弱的机器人。
- 可以让对手白白浪费能量，同时自己慢慢磨血。

### 6.2 shield 的成本

- 持续发射低威力子弹消耗能量，且击中返还能量只有 `3 × power`。
- 拦截失败时等于白送能量给对方。
- 对付高机动、低威力速射的对手收益有限。
- 一旦对手检测到你在 shield 并切换到边缘瞄准， shield 效率会大幅下降。

## 7. 什么时候值得做 shield

- 你的机器人有**稳定的 wave tracking** 和**精确的炮口控制**。
- 对手射速不快、威力偏高、弹道容易预测。
- 对手没有针对 shield 做反制（例如没有边缘瞄准）。
- 你希望用“生存”换取分数，而不是靠主动击杀。

## 8. 与 MechGarden 机器人的关系

MechGarden 当前的两个 1v1 机器人都没有把 bullet shield 作为核心战术：

- **Fencer**：以 wave surfing 和虚拟枪 GF 阵列为主，不主动 shield。
- **Ronin**：以 DC KNN 瞄准和自适应运动为主，同样不主动 shield；
  但内置了对敌方 shield 的检测与反制（`ShieldAimSelector`）。

如果你未来想做一个以 shield 为核心机制的机器人，需要至少具备：

1. 与 Fencer/Ronin 同等精度的 wave tracking。
2. 能同时处理多个来袭 wave 的拦截调度。
3. 对自身未来位置的准确预测。
4. 对敌方是否反制 shield 的在线判断。
